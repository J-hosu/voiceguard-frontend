# 03. 화면별 상세 설계

> 기준은 **Figma 이미지**입니다. 각 화면마다 **데이터 출처**를 명시했습니다:
> ✅ 실제 API/온디바이스로 가능 · 🟡 우회 구현 · 🔴 백엔드 없음(요청 필요, `05` 참조) · ⚪ 정적/로컬.

## 내비게이션 구조

```text
[온보딩/권한] (최초 1회)
   └─▶ 하단 탭: [홈] [히스토리] [설정(정보)]
        홈 ─(실시간 통화 감지)▶ 실시간 감지 → (임계값 도달) 위험 경고 모달
        홈 ─(음성 파일 분석)▶ 파일 업로드 → 분석 → 분석 결과 요약 ─▶ 근거 상세
        히스토리 ─(항목 탭)▶ 분석 결과 요약 ─▶ 근거 상세
        설정/정보 ─▶ 예방 정보 / 대응 방법
```

라우팅: `go_router`. 하단 탭 3개(홈/히스토리/정보). 결과/상세/모달은 push.

---

## 화면 1 — 온보딩 / 권한요청  (우선순위: 상)

- **구성**: 방패 로고 + "VoiceGuard AI" + 소개 문구 / "마이크 권한"·"알림 권한" 토글 / [권한 허용하고 시작하기] 버튼.
- **기능**: 최초 1회 노출(이후 스킵 플래그 `shared_preferences`). `permission_handler`로 `RECORD_AUDIO`, `POST_NOTIFICATIONS` 요청. 하단 안내: "음성 분석은 목적으로만 사용되며 기기와 서버에 안전하게 처리됩니다."
- **데이터 출처**: ⚪ 전부 로컬(권한/플래그).

## 화면 2 — 홈  (우선순위: 상)

- **구성**:
  - 앱바: "VoiceGuard AI" + 설정(⚙) 아이콘
  - 히어로: "이번 달 안전하게 지켜드렸어요 · **N건 위험 통화 탐지**"
  - 진입 버튼 2개: **[🎙 실시간 통화 감지]**(Primary/Filled), **[📁 음성 파일 분석]**(Outlined)
  - "최근 분석 내역"(전체보기 링크): 최근 3건 카드(유형·시각·위험도 %·색 배지)
  - 하단 탭
- **데이터 출처**:
  - 최근 3건: ✅ `GET /calls` 상위 3개
  - "N건" 월간 카운트: 🟡 `GET /calls`를 앱에서 이달·위험등급 필터 집계(전용 통계 API 없음, `05` 항목 6)

## 화면 3 — 음성 파일 업로드  (우선순위: 상, 단 제약 있음)

- **구성**: "여기로 파일을 올려주세요 (MP3·WAV·3~5분·50MB 이하)" 드롭존 + [파일 선택] / 업로드 상태(파일명·크기·진행률 %) / [분석 시작]. 하단 안내: "업로드 후 STT로 텍스트를 변환하고 위험도를 분석합니다. 30초~1분 정도 걸릴 수 있어요."
- **기능**: `file_picker`로 오디오 선택 → 분석 시작 → 결과 화면(7)로.
- **데이터 출처**: 🟡 **[확정: 온디바이스 STT 우회]** 백엔드에 파일/STT 엔드포인트가 없으므로 백엔드 요청 없이 앱에서 처리한다.
  - 흐름: 선택한 파일을 앱이 온디바이스 STT로 전사 → 전사된 발화를 WS `start`+`message`로 순차 전송 → 화면 4~7과 **동일 파이프라인 재사용**.
  - ⚠️ 구현 리스크: 안드로이드 `SpeechRecognizer`는 마이크 실시간 입력 기준이라 **파일 재생 오디오를 직접 전사하기 까다롭다.** 안정화 방법:
    (a) 파일을 스피커로 재생하며 마이크로 재입력(가장 단순, 데모용), 또는
    (b) `speech_to_text`가 파일 소스를 못 받으면, 데모 안정성을 위해 미리 준비한 전사 텍스트를 시간축에 맞춰 재생하는 폴백 경로를 둔다.
  - "업로드 진행률 %"는 이 경로에선 **전사 진행률**로 의미를 재정의.
  - (향후 고도화) 서버 파일 STT가 생기면 교체 — 지금은 요청하지 않음(`05` 항목 1 참고, 우선순위 하향됨).

## 화면 4 — 실시간 통화 감지  (우선순위: 상)

- **구성(Figma)**: 통화 화면처럼 보이는 UI(발신번호, 통화시간, 녹음/스피커/음소거/키패드 버튼) 위에 **VoiceGuard AI 검사 오버레이 배지**("73 · 주의 · 저금리 대출·신용평점 언급").
- **중요 플랫폼 제약**: 실제 통신사 통화 오디오를 가로채는 것은 **Android 정책상 불가**(기능명세서 §13). 이 화면은 **앱 자체의 스피커폰+마이크 감지 세션**을 통화 UI 스타일로 보여주는 데모입니다. 실제 전화앱 위에 그리는 시스템 오버레이가 아닙니다.
- **기능**: 시작 시 STT 리스닝 루프 + WS 세션(`start`). 발화 인식마다 `message` 전송, 응답으로 상단 배지의 위험도/등급/키워드 실시간 갱신. 임계값 도달 시 화면 6 자동 노출.
- **데이터 출처**: ✅ 온디바이스 STT + WS. 배지 위험도=`risk_score`, 키워드=`matched_patterns`.

## 화면 5 — 실시간 대화 내용  (우선순위: 상)

- **구성(Figma)**: 상단 "실시간 대화 내용 · 번호 · 통화 중 · ●REC" / 노란 요약 배지("73 · 주의 · 대출빙자형 의심 · 위험 키워드 3건 감지") / **채팅형 전사 로그**(상대방=좌측 회색, 나=우측 파랑, 각 말풍선에 타임스탬프 00:04 등, 일부 단어 하이라이트) / 하단 "AI가 실시간으로 대화를 분석하고 있습니다…".
- **기능**: 화면 4와 같은 세션의 다른 뷰(탭 전환 or 스크롤). STT 최종결과가 나올 때마다 말풍선 추가, WS 응답의 위험도로 상단 배지 갱신.
- **데이터 출처**:
  - 전사·타임스탬프: ✅ 온디바이스 STT(타임스탬프는 앱이 로컬 기록)
  - 턴별 위험도/의심 판정: ✅ WS 응답
  - 말풍선 내 **단어 단위 하이라이트**: 🔴 백엔드가 단어 위치를 주지 않음. `matched_patterns` 카테고리만 있음 → 앱에서 키워드 사전 매칭으로 근사 하이라이트하거나 백엔드 요청(`05` 항목 4)
  - 화자 좌/우 구분: 🟡 온디바이스 화자분리 불가 → role 규칙 확인 전까지 단일 화자 기준(`05` 항목 7)

## 화면 6 — 위험 경고 모달  (우선순위: 상, 핵심 차별화)

- **구성(Figma)**: 전면 빨강 배경, 대형 ⚠ 아이콘, "보이스피싱 위험 감지!", "위험도 88% · 강한 경고", **탐지 근거** 리스트(수사기관 사칭 표현 / 계좌 범죄 연루 언급 / 금전 이체 유도 정황), "통화를 끊고 해당 기관의 공식 대표번호로 직접 확인하세요", 버튼 **[대처 방법]**·**[계속 지켜보기(닫기)]**.
- **기능**: `risk_score ≥ 0.85`(강한 경고) 시 자동 풀스크린. **진동 + 알림음** 동시(`vibration` + `flutter_local_notifications`). 접근성상 색+아이콘+텍스트 라벨 병행.
- **데이터 출처**:
  - 위험도·근거: ✅ WS `phishing_detected`(`risk_score`, `matched_patterns`, `core_evidence`)
  - **권장 행동 문구**: 🔴 WS 응답에 없음 → 유형별 정적 매핑(`info` feature의 대응 가이드 재사용) 또는 백엔드 요청(`05` 항목 11)
  - [대처 방법] → 화면 "대응 방법"으로 이동

## 화면 7 — 분석 결과 요약  (우선순위: 상)

- **구성(Figma)**: 빨강 헤더 "분석 결과 · 보이스피싱 의심 · **84%** · 수사기관 사칭형" + 공유 아이콘 / 탭 **[요약][대화 내용][탐지 근거]** / "핵심 근거" 항목 3개(범죄 연루 언급/안전계좌 이체 요구/비밀 유지 압박) / "권장 행동" 박스 / 하단 탭.
- **기능**: 통화 종료 후 요약. [요약]=핵심 근거+권장행동, [대화 내용]=화면 5 로그, [탐지 근거]=화면 8.
- **데이터 출처**:
  - 위험도·유형·핵심근거: ✅ 세션 중 수신한 WS `phishing_detected`(최종/최고 점수 이벤트) 사용
  - **과거 통화를 히스토리에서 열 때**: 🔴 단일 통화 상세 API(`GET /calls/{id}`) 없음 → 세션 중 앱 메모리에 저장해 둔 것만 요약 가능. 앱 재시작 후 과거 통화 상세는 불가 → 로컬 DB 저장 or 백엔드 요청(`05` 항목 3)
  - 권장 행동: 🔴 위 화면 6과 동일(정적 매핑/요청)
  - CSV 다운로드/공유: 🟡 앱에서 로컬 CSV 생성(`05` 항목 12), 공유 `share_plus`

## 화면 8 — 근거 상세  (우선순위: 상)

- **구성(Figma)**: 날짜 헤더 / "범죄 유형" 카드("해당 통화는 '대출 빙자형'으로 의심됩니다 · 주의 73") / "의심되는 주요 단어" **칩들**([강제상환][신용평점][우회]) / "주요 단어를 사용한 문장"(해당 단어가 강조된 설명 문단).
- **데이터 출처**:
  - 유형·위험도: ✅ WS 응답
  - **의심 단어 칩(단어 단위)**: 🔴 백엔드는 카테고리(`matched_patterns`)만 제공, 개별 단어 목록 없음 → 앱 키워드 사전 매칭 근사 or 백엔드 요청(`05` 항목 4)
  - **단어가 쓰인 문장 + 하이라이트**: 🔴 `core_evidence` 문장 1개만 있음, 단어↔문장 매핑 없음 → 요청(`05` 항목 4)

## 히스토리 목록  (우선순위: 중)

- **구성(Figma)**: 상단 필터 카운트 탭 **[위험 7][주의 3][정상 18]** / 카드 리스트(유형·날짜·소스(실시간/파일)·위험도 %·색 스트라이프+배지). 탭하면 화면 7로.
- **데이터 출처**:
  - 목록: ✅ `GET /calls` (단, 필드 스키마 확인 필요 — `05` 항목 2)
  - 필터 카운트(위험/주의/정상): 🟡 목록을 앱에서 등급별 집계
  - 검색/필터: ⚪ 클라이언트 측

## 설정  (우선순위: 중)

- **구성**: 경고 임계값 슬라이더(시연 조정용, 70~85%), 다크모드 토글, 큰 글씨 모드, 앱 정보/개인정보 처리방침.
- **데이터 출처**: ⚪ 전부 `shared_preferences` 로컬. 임계값은 `risk_level.dart` 계산에 주입.

## 예방 정보 안내  (우선순위: 중하, 여유 시)

- **구성(Figma)**: "알아두면 당하지 않습니다" / 3대 유형 카드(수사기관 사칭형 / 대출사기형 / 납치·협박형) / "3초 대응 원칙" / [금융감독원 신고 1332] 버튼.
- **데이터 출처**: ⚪ 정적 콘텐츠 + 전화 인텐트(1332). 금감원 공식 링크.

## 대응 방법 안내(정보)  (우선순위: 중하)

- **구성(Figma)**: "지금 의심 통화 중이라면" 3단계(개인정보·계좌·인증번호 말하지 않기 / 전화 끊고 기관에 직접 재확인 / 이미 송금했다면 은행 지급정지 요청) / "유형별 대응 방법"(기관 사칭형·대출 빙자형 …).
- **데이터 출처**: ⚪ 정적. 화면 6의 [대처 방법]과 권장행동 매핑을 공유.
