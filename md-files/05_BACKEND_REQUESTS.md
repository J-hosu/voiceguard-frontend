# 05. 백엔드 요청 사항 (프론트 필요 ⨯ 백엔드 없음)

> Figma·기능명세서가 요구하지만 **실제 백엔드(`API_SPEC.md`)에는 없는** 항목입니다.
> 각 항목마다 **① 왜 필요한지 ② 프론트 우회 가능 여부 ③ 백엔드 요청안**을 정리했습니다.
> 우선순위: 🔴 필수(핵심 화면 블로킹) · 🟠 중요 · 🟡 있으면 좋음.

---

## 🟡 1. 음성 파일 업로드 + 서버 STT 엔드포인트  〔결정: 당장 요청 안 함 — 온디바이스 우회〕

- **필요 화면**: 화면 3(음성 파일 분석).
- **현황**: 파일 업로드/파일 STT 엔드포인트 없음. WS는 텍스트만 받음.
- **결정**: **온디바이스 STT로 우회**하기로 확정(백엔드 요청 없이 진행). 상세 구현/리스크는 `03_SCREENS.md` 화면 3 참조.
- **아래는 향후 서버 STT로 전환할 경우의 참고 요청안**(지금은 보류):
  ```text
  POST /calls/upload  (multipart/form-data, file)
    → { "call_id": <int>, "status": "uploaded" }
  POST /calls/{call_id}/transcribe   (서버가 STT+분석 수행)
    → { "call_id": <int>, "status": "transcribing" }
  ```
  또는 최소한 "오디오 청크 → STT 텍스트" 변환 엔드포인트라도.

## 🔴 2. `GET /calls` 응답 스키마 명세

- **필요 화면**: 히스토리, 홈 최근내역/월간 카운트.
- **현황**: 엔드포인트는 있으나 **응답 필드가 문서화돼 있지 않음.**
- **요청안**: 각 항목에 아래 필드 포함 여부/필드명 확정.
  ```jsonc
  {
    "id": 1,
    "name": "테스트 통화",
    "category": "수사기관 사칭형",  // 유형 라벨
    "risk_score": 0.84,            // 0~1
    "risk_level": "high",          // low/medium/high
    "source": "realtime",          // realtime | file
    "created_at": "2025-01-01T14:20:00Z"
  }
  ```
  - 페이지네이션/정렬(최신순) 지원 여부도 확인.

## 🔴 3. 단일 통화 상세 조회 `GET /calls/{id}` (전체 발화 + 근거 포함)

- **필요 화면**: 화면 7·8, 히스토리 항목 탭 시 상세.
- **현황**: 단일 통화 상세 API 없음. 통화 종료/앱 재시작 후 과거 통화의 전사·근거를 다시 불러올 방법이 없음.
- **프론트 우회**: 세션 동안 앱 메모리(또는 로컬 DB `sqflite`)에 발화·이벤트를 저장. 단, 서버 재시작/기기 변경 시 유실.
- **요청안**:
  ```jsonc
  GET /calls/{id}
  → {
      "id": 1, "name": "...", "category": "...",
      "final_score": 0.86, "risk_level": "high",
      "turns": [
        { "turn_index":1, "role":"speaker_a", "text":"...",
          "risk_score":0.2, "matched_patterns":[], "core_evidence":null }
      ],
      "summary": "...", "recommended_action": "..."
    }
  ```

## 🟠 4. 근거 상세: 의심 단어(단어 단위) + 단어↔문장 매핑/하이라이트

- **필요 화면**: 화면 8(의심되는 주요 단어 칩, 단어가 쓰인 문장 강조), 화면 5(말풍선 내 단어 하이라이트).
- **현황**: 백엔드는 **카테고리**(`matched_patterns`)와 근거 **문장 1개**(`core_evidence`)만 제공. 개별 위험 단어 목록도, 어느 문장의 어느 단어인지도 없음.
- **프론트 우회**: 앱이 자체 키워드 사전으로 근사 매칭·하이라이트(정확도 한계).
- **요청안**: `phishing_detected`(및 상세 API)에 아래 추가.
  ```jsonc
  "risk_keywords": ["강제상환", "신용평점", "우회"],
  "evidence": [
    { "keyword":"신용평점", "sentence":"신용 평점이 낮아 대출 연장이 어렵습니다.",
      "spans":[[3,7]] }   // 하이라이트용 문자 오프셋(있으면 최상)
  ]
  ```

## 🟠 5. 위험도 등급 기준 정렬 확인

- **필요 이유**: 3개 문서 기준이 다름 —
  API(`low<0.45 / medium 0.45~0.75 / high≥0.75`) vs 기능명세서(`주의 70% / 강한경고 85%`) vs Figma(84%=위험).
- **프론트 처리(잠정)**: UI는 기능명세서 70/85 기준으로 `risk_score`에서 직접 계산, 백엔드 `risk_level`은 참고용.
- **요청안**: (a) UI 라벨/경고 트리거를 70/85로 확정해도 되는지, (b) 백엔드 `risk_level` 문자열을 UI에서 신뢰해야 하는지, (c) Figma의 84%=빨강/"보이스피싱 의심" 표기 시점 확정.

## 🟠 6. 월간/기간 통계 (홈 "N건 위험 통화 탐지")

- **현황**: 집계 엔드포인트 없음.
- **프론트 우회**: `GET /calls`를 받아 앱에서 이달·위험등급 필터 카운트(데이터 많아지면 비효율).
- **요청안(선택)**: `GET /calls/stats?period=month` → `{ "danger": 12, "warning": 3, "safe": 18 }`.

## 🟡 7. WS `role` 값 규칙

- **현황**: 예시에 `speaker_a`만 등장. 사용자 본인 발화/상대 발화 구분 규칙 불명.
- **프론트 상황**: 온디바이스 화자분리 불가. 캡처 발화를 단일 role로 보낼 예정.
- **요청안**: `role` 허용값과 의미(예: `speaker_a`=상대, `speaker_b`=본인) 명시. 단일 role 전송이 분석에 문제없는지 확인.

## 🟡 8. WS `start`의 `device_id` 발급 방법

- **현황**: `start`에 `device_id` 필수인데 발급/등록 절차가 없음.
- **프론트 상황**: MVP는 고정값(예: 1) 사용 예정.
- **요청안**: 고정값 사용 가능 여부, 또는 기기 등록 엔드포인트 필요 여부.

## 🟡 9. WS 세션 재연결/재개 규칙

- **현황**: 연결 끊김 시 재개 규칙 없음. 재연결 후 `start`하면 새 call로 취급될 가능성.
- **요청안**: 기존 `call_id`로 세션 재개(resume) 지원 여부, 미지원 시 권장 처리.

## 🟡 10. `phishing_detected.notification` 스키마

- **현황**: 빈 객체 `{}`로만 예시됨.
- **요청안**: 실제 필드 정의(제목/본문/채널 등) 또는 "미사용" 확정.

## 🟡 11. 권장 행동(recommended_action) 텍스트

- **필요 화면**: 화면 6·7의 "권장 행동".
- **현황**: WS 응답에 없음(기능명세서 §4.6 LLM 출력엔 있었으나 실제 API엔 미포함).
- **프론트 우회**: 유형별 권장행동을 앱에 정적 매핑(대응 방법 화면 콘텐츠 재사용).
- **요청안(선택)**: `phishing_detected`에 `recommended_action` 문자열 포함.

## 🟡 12. 결과 내보내기(CSV/JSON)

- **필요 화면**: 화면 6/7 [CSV 다운로드].
- **현황**: 내보내기 엔드포인트 없음.
- **프론트 우회**: 앱이 보유 데이터로 로컬 CSV 생성 후 `share_plus` 공유(권장, 요청 불필요할 수 있음).

## 부록: "실제 통화 앱 연동"은 백엔드 문제가 아님 (참고)

Figma 화면 4의 통화 UI는 실제 통신사 통화 오디오 가로채기가 아니라 **앱의 스피커폰+마이크 데모**입니다. Android 정책상(Accessibility/CallScreening 제약, 기능명세서 §13) 실제 통화 오디오 실시간 수집은 MVP 범위 밖입니다. 백엔드 요청 사항이 아니라 **프론트 구현 방식(스피커폰 데모)으로 처리**합니다.
